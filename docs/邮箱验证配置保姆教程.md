# 邮箱验证配置保姆教程

这份文档只讲一条路线：

- 发信服务：`Resend`
- 域名 DNS：`阿里云域名解析`
- 你当前项目域名：`virtuals.club`
- 项目当前访问地址：`http://virtuals.club`

不要自己发挥，不要改成别的邮箱平台，不要改成别的子域名。  
照着这份文档做完，就能让项目发送“注册验证邮件”。

---

## 1. 这次到底要配置什么

你现在项目里的这几项配置，作用分别是：

- `APP_PUBLIC_BASE_URL`
  - 告诉系统：用户点邮件里的验证链接时，应该跳回哪个网站地址
- `EMAIL_ENABLED`
  - 是否开启邮箱验证功能
- `EMAIL_SMTP_HOST`
  - 邮件服务器地址
- `EMAIL_SMTP_PORT`
  - 邮件服务器端口
- `EMAIL_SMTP_USERNAME`
  - 邮件服务器用户名
- `EMAIL_SMTP_PASSWORD`
  - 邮件服务器密码
- `EMAIL_SMTP_USE_TLS`
  - 是否启用加密发信
- `EMAIL_FROM_ADDRESS`
  - 用户实际看到的发件邮箱
- `EMAIL_FROM_NAME`
  - 用户实际看到的发件人名称
- `EMAIL_VERIFY_TOKEN_TTL_SEC`
  - 验证链接过期时间，单位是秒

你这次真正要去“获得”的，只有这 4 样：

1. `Resend API Key`
2. `Resend 域名验证通过`
3. `发件邮箱地址`
4. `网站访问地址`

---

## 2. 这次固定采用的最终配置

这次不要自己想名字，直接用下面这组：

- 用来发邮件的子域名：`mail.virtuals.club`
- 发件邮箱：`noreply@mail.virtuals.club`
- 发件人名称：`Virtuals Whale Radar`
- 网站地址：`http://virtuals.club`

所以你最终写进 `config.json` 的值，固定按下面这组来：

```json
{
  "APP_PUBLIC_BASE_URL": "http://virtuals.club",
  "EMAIL_ENABLED": true,
  "EMAIL_SMTP_HOST": "smtp.resend.com",
  "EMAIL_SMTP_PORT": 587,
  "EMAIL_SMTP_USERNAME": "resend",
  "EMAIL_SMTP_PASSWORD": "这里替换成你的 Resend API Key",
  "EMAIL_SMTP_USE_TLS": true,
  "EMAIL_FROM_ADDRESS": "noreply@mail.virtuals.club",
  "EMAIL_FROM_NAME": "Virtuals Whale Radar",
  "EMAIL_VERIFY_TOKEN_TTL_SEC": 1800
}
```

注意：

- `APP_PUBLIC_BASE_URL` 现在先用 `http://virtuals.club`
- 以后如果你把 HTTPS 配好了，再改成 `https://virtuals.club`
- 这次不要先改成 HTTPS

---

## 3. 第一步：注册 Resend 账号

打开：

- [https://resend.com](https://resend.com)

按页面提示注册并登录。

登录成功后，你后面只做 2 件事：

1. 添加域名
2. 创建 API Key

---

## 4. 第二步：在 Resend 里添加发信域名

### 4.1 进入 Domains 页面

登录 Resend 后，进入：

- `Domains`

然后点击：

- `Add Domain`

### 4.2 输入域名

这一步不要填 `virtuals.club`，也不要填 `www.virtuals.club`。  
这次固定填：

```text
mail.virtuals.club
```

填完后提交。

### 4.3 记住一个原则

Resend 会给你几条 DNS 记录。  
这些记录的内容，**你不要自己编，不要自己改，不要凭感觉写**。

你只做一件事：

- 把 Resend 页面里给你的每一条记录，原样抄到阿里云 DNS

---

## 5. 第三步：去阿里云给 `mail.virtuals.club` 加 DNS 记录

### 5.1 打开域名解析页面

你已经在阿里云里管理 `virtuals.club` 域名。

进入阿里云域名解析控制台，找到：

- `virtuals.club`

进入 DNS 设置页面。

### 5.2 你要添加什么

此时回到 Resend 的 `mail.virtuals.club` 域名详情页。  
Resend 会显示几条 DNS 记录，通常会包含这些类型中的几种：

- `TXT`
- `MX`
- `CNAME`

你只需要在阿里云里：

- 点击 `添加记录`
- 一条一条照抄

### 5.3 复制时的规则

每一条都按 Resend 页面原样填写：

- 记录类型：照抄
- 主机记录：照抄
- 记录值：照抄
- MX 优先级：如果有就照抄
- TTL：保持默认即可

不要自己做这些事情：

- 不要把 `mail.virtuals.club` 改成 `virtuals.club`
- 不要删掉点号
- 不要补多余空格
- 不要把某一条 TXT 拆开
- 不要只加一部分记录

### 5.4 加完后的正确状态

加完以后，回到 Resend 域名页面，点击：

- `Verify DNS Records`

然后等待。

DNS 生效有时很快，有时要几分钟到几小时。  
这时候你要做的不是猜，而是等 Resend 状态变成：

- `Verified`

只有看到 `Verified`，这一步才算完成。

---

## 6. 第四步：在 Resend 创建 API Key

### 6.1 打开 API Keys 页面

在 Resend 后台，进入：

- `API Keys`

点击：

- `Create API Key`

### 6.2 创建方式

名字你可以直接填：

```text
Virtuals Whale Radar SMTP
```

创建成功后，页面会显示一串以 `re_` 开头的密钥。

例如：

```text
re_xxxxxxxxxxxxxxxxx
```

你要做的是：

- 立刻复制
- 保存到记事本

因为这个值后面就要填到：

- `EMAIL_SMTP_PASSWORD`

注意：

- 这个值不是邮箱密码
- 它就是 Resend 的 API Key
- 在 SMTP 配置里，它充当“密码”

---

## 7. 第五步：在服务器里修改 `config.json`

### 7.1 登录你的服务器

进入服务器终端后，执行：

```bash
cd /opt/virtuals-whale-radar
```

### 7.2 先确认文件在不在

执行：

```bash
ls
```

你应该能看到：

```text
config.json
```

### 7.3 用 nano 打开配置文件

执行：

```bash
nano config.json
```

### 7.4 找到并修改这些字段

把下面这些值写进去，或者改成下面这样：

```json
"APP_PUBLIC_BASE_URL": "http://virtuals.club",
"EMAIL_ENABLED": true,
"EMAIL_SMTP_HOST": "smtp.resend.com",
"EMAIL_SMTP_PORT": 587,
"EMAIL_SMTP_USERNAME": "resend",
"EMAIL_SMTP_PASSWORD": "这里替换成你的 Resend API Key",
"EMAIL_SMTP_USE_TLS": true,
"EMAIL_FROM_ADDRESS": "noreply@mail.virtuals.club",
"EMAIL_FROM_NAME": "Virtuals Whale Radar",
"EMAIL_VERIFY_TOKEN_TTL_SEC": 1800,
```

你只需要替换一项：

- `EMAIL_SMTP_PASSWORD`

它要换成你刚刚在 Resend 创建的那串 `re_...`

### 7.5 保存文件

在 `nano` 里：

1. 按 `Ctrl + O`
2. 按回车
3. 按 `Ctrl + X`

---

## 8. 第六步：重启项目服务

保存好 `config.json` 后，在服务器里执行下面这组命令：

```bash
systemctl restart vwr@writer
systemctl restart vwr@realtime
systemctl restart vwr@backfill
```

如果你也想顺手重启 `SignalHub`，可以再执行：

```bash
systemctl restart vwr-signalhub.service
```

### 8.1 检查服务状态

继续执行：

```bash
systemctl status vwr@writer --no-pager
systemctl status vwr@realtime --no-pager
systemctl status vwr@backfill --no-pager
```

你看到每个服务里出现：

```text
active (running)
```

就说明服务已经正常起来了。

## 9. 第七步：实际测试邮箱验证

现在去网站：

- [http://virtuals.club/auth/register](http://virtuals.club/auth/register)

按下面顺序测试：

1. 注册一个新邮箱
2. 页面应该提示：验证邮件已发送
3. 打开这个邮箱
4. 点开验证链接
5. 页面应该自动进入 `/app`
6. 登录后应该能看到 `20` 积分到账

如果这 6 步都成功，说明整个邮箱验证流程已经打通。

## 10. 你最容易犯错的地方

### 错误 1：把根域名拿去做发信域名

不要这样做：

```text
virtuals.club
```

这次固定用：

```text
mail.virtuals.club
```

### 错误 2：自己乱写 DNS 记录

不要自己猜记录值。  
必须照抄 Resend 页面给出的内容。

### 错误 3：端口填成 465

你当前项目代码用的是：

- SMTP + TLS 升级方式

所以这次固定填：

```text
587
```

不要填 `465`。

### 错误 4：`EMAIL_FROM_ADDRESS` 和验证过的域名不一致

如果你验证的是：

```text
mail.virtuals.club
```

那发件邮箱就必须写成：

```text
noreply@mail.virtuals.club
```

不要写成：

```text
noreply@virtuals.club
```

### 错误 5：Resend 还没显示 Verified 就开始测试

如果 Resend 域名状态还不是 `Verified`，先不要测试注册。  
否则邮件大概率发不出去。

## 11. 这几个配置分别从哪里来

最后再用一句话说清楚：

- `APP_PUBLIC_BASE_URL`
  - 你的项目对外访问地址
  - 当前固定写：`http://virtuals.club`

- `EMAIL_SMTP_HOST`
  - 来自 Resend 官方 SMTP 文档
  - 固定写：`smtp.resend.com`

- `EMAIL_SMTP_PORT`
  - 这次固定写：`587`

- `EMAIL_SMTP_USERNAME`
  - 来自 Resend 官方 SMTP 文档
  - 固定写：`resend`

- `EMAIL_SMTP_PASSWORD`
  - 你在 Resend 后台创建的 API Key

- `EMAIL_FROM_ADDRESS`
  - 你自己定，但必须属于已经在 Resend 验证通过的域名
  - 这次固定写：`noreply@mail.virtuals.club`

- `EMAIL_FROM_NAME`
  - 你自己展示给用户看的名字
  - 这次固定写：`Virtuals Whale Radar`

- `EMAIL_VERIFY_TOKEN_TTL_SEC`
  - 你自己定过期时间
  - 这次固定写：`1800`
