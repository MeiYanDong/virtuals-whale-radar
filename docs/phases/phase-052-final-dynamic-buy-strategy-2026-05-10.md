# Phase 052 Final Dynamic Buy Strategy

本文件整理 2026-05-10 围绕 98min 税率项目自动买入策略的讨论结论、被否掉的策略、当前采用策略和 SR/ISC 数据表现。

本阶段冻结策略口径；真实交易执行链路进入 Phase 053 分层测试。

## 1. 讨论过的策略

### 1.1 固定 50V + 300V 上限

口径：
- 硬条件：榜单 V、税率、榜单人数、成本样本、含税 FDV <= 榜单成本。
- 每次买入 `50 VIRTUAL`。
- 项目总投入上限 `300 V`。
- 同一税率档位最多买一次。

结果：
- SR 表现可用，能覆盖早期低 FDV 机会。
- ISC 会因为项目早期一路下跌而较快打满上限，后续更低 FDV 阶段无法继续买。

结论：
- 作为保守资金上限模型有价值。
- 不作为当前主策略，因为会在 ISC 这类一般项目上过早耗尽预算。

### 1.2 不限制买入上限 + 每次固定 50V

口径：
- 去掉 `300V` 项目最大投入。
- 每次仍固定买入 `50 VIRTUAL`。
- 同一税率档位最多买一次。

结果：
- SR 绝对收益更高。
- ISC 买入次数和总投入明显放大，暴露出“越跌越买”的资金占用问题。

结论：
- 只适合做压力测试。
- 不适合作为生产候选，因为单项目风险无法受控。

### 1.3 等榜单 V 暴增或 FDV 快速反弹后加仓

口径：
- 试图用“榜单 V 暴增 / FDV 反弹”确认项目变热，再加仓。

关键问题：
- SR 的 `21k -> 127k` 榜单 V 暴增被看到时，含税 FDV 已经从约 `219` 万跳到 `331` 万左右。
- 这个信号是机会被重新定价后的结果，不是低位买入前可用的预测信号。
- 只靠链上已发生数据，无法提前知道项目会像 SR 一样变热，还是像 ISC 一样继续走弱。

结论：
- 不采用。
- 不能用未来热度确认去解释过去买点，后续回测必须避免未来函数。

### 1.4 连续两个税率分钟买入后再横盘暂停

口径：
- 连续两个税率分钟都发生买入后；
- 如果下一税率分钟 FDV 相对上一买点变化 `<=10%`，跳过该税率档。

结果：
- SR `tax<=94`：买入 `5` 次，投入 `125V`，end ROI `+63.5692%`。
- ISC `tax<=89`：买入 `19` 次，投入 `475V`，end ROI `+38.5483%`。

结论：
- 已测试，但不作为当前最终策略。
- 相比 after1，它绝对 PnL 更高，但总投入更大，资金占用和误买风险更高。

## 2. 当前采用策略

策略名称：

`dynamic_25v_dip20_after1_flat10_no_cap`

### 2.1 硬条件

每个买点必须同时满足：

- 项目处于 `live`。
- 榜单累计投入 `boardSpentV >= 5,000 V`。
- 榜单人数 `whaleRows >= 20`。
- 成本样本 `costRows >= 5`。
- 当前税率 `buyTaxRate <= rule threshold`。
- 含税估算 FDV `estimatedFdvWanUsdWithTax <= boardCostWanUsd`。
- 同一税率档位最多买一次。

### 2.2 单笔买入数量

- 默认每次买入 `25 VIRTUAL`。
- 如果当前含税估算 FDV `<= 我方历史加权买入 FDV * 0.8`，本次买入 `50 VIRTUAL`。
- 我方历史加权买入 FDV：

```text
sum(每次买入 V * 当时含税估算 FDV) / sum(买入 V)
```

当前 SR/ISC 回测中，`50V` 加倍条件均未触发。

### 2.3 暂停规则

- 某税率分钟发生买入后；
- 到下一税率分钟时，当前含税估算 FDV 相对上一笔买入 FDV 的变化幅度 `<= 10%`；
- 则跳过该税率分钟；
- 再下一个税率分钟重新开始计算。

这里的“分钟”不是自然分钟整点，而是 Virtuals 税率 tick。税率 tick 以官网 `launchedAt` 为锚点。

### 2.4 当前边界

- 当前回测不限制项目最大买入上限。
- 这不等于生产自动买入开关。
- 下一步进入 Phase 053 execution pipeline：先做 realtime dry-run would-buy，再依次测试交易构造、模拟、签名、小额主网 canary 和自动执行灰度。
- 当前文档不代表真实交易已启用；真实广播必须经过 Phase 053 全链路验收。

## 4. 当前结论

- SR 的低位机会无法靠“看到榜单 V 暴增后再确认”捕捉；必须在硬条件首次满足时就小额入场。
- ISC 证明早启动会增加绝对 PnL，但会降低 ROI，并明显放大总投入。
- 当前策略用 `25V` 基础仓降低早买风险，用 after1 横盘暂停减少连续重复买入和资金占用。
- `50V` 加倍规则当前没有触发，后续若要继续优化，可以单独测试“低于历史最高买入成本 20%”而不是“低于我方加权成本 20%”。
- 当前策略进入 Phase 053：先跑 realtime dry-run would-buy，再分阶段测试 order builder、tx simulator、local signer、小额主网 canary 和自动执行灰度。
